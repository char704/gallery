## Remaining Priority Issues

### P2: Album Context Is Still Absent

The prompt asks for album metadata where available, but the current `Photo` type and photo detail API response do not expose album membership. The UI should not invent this information. A future backend-supported enhancement could add album context safely.

### P2: Modal Browser Verification Was Limited

The modal code path is covered by tests, but the in-app browser timed out while navigating from Explore to the modal route. A future manual pass should verify the actual modal at desktop and mobile sizes.

### P3: Shared Form Styling Is Still Repeated

Photo Detail and Photo Viewer now use consistent fields, but the same field/button class patterns are repeated. This should be considered during the shared application shell or final consistency pass, not extracted prematurely here.
